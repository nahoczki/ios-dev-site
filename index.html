The Evolution iOS Development
1996
Acquiring Objective-C
Apple Inc. acquires Objective-C; the primary programming language used to build the initial iPhone Operating System (iOS)
2007
Start of the iPhone
iOS is a mobile operating system developed by Apple Inc. It was first released as iPhone OS in June 2007.
2008
UIKit Release
UIKit, the most popular way of creating the user interface of iOS apps gets released to the public
2014
Swift Release
Swift is released to replace Objective-C and to be the best available language for uses ranging from systems programming, to mobile and desktop apps, scaling up to cloud services
2019
SwiftUI Release
Apple introduced SwiftUI in 2019 and the framework has been evolving at a rapid pace ever since. Unlike UIKit and AppKit, SwiftUI is a cross-platform framework. With SwiftUI, Apple offers developers a solution for rapidly building applications.
2023+

Event Driven iOS Development
Event-driven programming is a programming paradigm in which the flow of a program is determined by events, such as user input or system notifications. In event-driven programs, the program typically begins by setting up event listeners, which are functions that are called in response to a specific event. When an event occurs, the program's event listeners are triggered and the appropriate code is executed. * Interact with the phone!

User Interface iOS Development
Both iOS and Android have their own unique user interface that is designed to make it easy for users to interact with their devices. * Hover over the circles to the right for more info on each element! How's it made?
Title Positioning iOS centers their titles in the navigation bar while Android aligns to the left
Fonts iOS uses SF (San Francisco) while Android utilizes Roboto
Cards When designing cards, iOS keeps it simple with rounded corners while Android does not round corners and adds a drop shadow.
Icons iOS and Android use a different icon set. iOS opts in to use more rounded icons while Android sticks to "squared" icons
Buttons iOS keeps their button simple with a filled version and a "just text" version while Android uses similar methods to their cards but has a outlined version. Material Design also tends to keep buttons capitalized.
