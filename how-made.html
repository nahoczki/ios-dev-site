UIKit & Storyboards How’s it made
Storyboards A storyboard is a file that allows a developer to create UIs with drag-and-drop and premade elements
Elements and Constraints When dragging elements onto the screen the app will need a sense of knowing where the element is relative to the screen
Connecting the elements to code With storyboards, as a developer you are able to “wire” the elements straight to the swift code and be able to access its contents and modify it’s properties.
SwiftUI How’s it made
SwiftUI How’s it made
SwiftUI Apple introduced a new way of creating app UIs with SwiftUI. SwiftUI allows a developer to programmatically create elements instead of using a storyboard
Elements and Constraints Unlike storyboards, when you use SwiftUI you will not need to worry about constraints. SwiftUI provides you with elements such as VStack and HStack, Vertical Stack and Horizontal Stack, to “auto layout” your app’s elements
Connecting the elements to code Since you’re writing code, you don’t have to connect your elements to anything. Everything runs off of the concept of “state” where when a state variable changes the UI will update accordingly
Alternatives How’s it made

Flutter is a mobile app development framework created by Google. It uses the Dart programming language and allows developers to build natively compiled applications for mobile, web, and desktop. Flutter has a fast development cycle with hot reload, a rich set of customizable widgets, and integrated testing support. It also has excellent documentation and a large, active community. Flutter is suitable for creating both simple and complex applications and has been used by companies such as Alibaba and Google Ads.

Xamarin is a cross-platform mobile app development platform that allows developers to build native iOS, Android, and Windows applications using C# and the .NET framework. Xamarin has strong integration with Visual Studio, providing a familiar development environment for C# developers. It also has excellent documentation and a large community of users. Xamarin is suitable for building a wide range of applications, from simple utilities to complex, data-intensive apps. It has been used by companies such as Siemens and Alaska Airlines.

React Native is a mobile app development framework created by Facebook. It allows developers to build natively compiled applications for mobile, using JavaScript and the React framework. React Native has a fast development cycle with hot reloading, a large library of components, and support for third-party plugins. It also has a large, active community and excellent documentation. React Native is suitable for building a wide range of applications, from simple utilities to complex, data-intensive apps. It has been used by companies such as Facebook, Instagram, and Airbnb.

Pros:

  • - Native iOS development allows for the creation of high-performance and visually stunning applications that take full advantage of the hardware and software features of iOS devices.
  • - It offers a high level of control over the look and feel of the app, as well as its functionality.
  • - Native iOS development typically results in applications that are more reliable and stable compared to alternatives.

Cons:

  • - Native iOS development requires a relatively steep learning curve, as it involves mastering complex development tools and languages.
  • - It can be time-consuming and resource-intensive, as separate codebase and development efforts are needed for each platform (iOS, Android, etc.)
